# Архитектура DB

```text
WPF / внешний клиент
        │
        ▼
C API db_native
        │
        ▼
TCP protocol: len + SQL / status + len + JSON
        │
        ▼
customdb_server
        │
        ▼
SqlEngine ── CatalogStore ── data/*.store
```

## Особенности реализации

- Сервер создаёт отдельный поток на каждое клиентское подключение.
- Глобальное состояние активной базы сохраняется в хранилище, поэтому последовательные короткие подключения могут выполнять `USE db;`, а затем отдельные запросы к выбранной базе.
- Parser не использует кодогенерацию и внешние SQL-библиотеки: поддерживаемые команды разбираются маленьким hand-written parser.
- Значения в строках хранятся как текст, а типы используются для валидации при вставке и обновлении.
